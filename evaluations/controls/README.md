# Control arm

The control is the treatment's paired negative control. It uses exactly the same model ID and version, trial count, tools, tool permissions, fixture files, working directory, and `task.md`. It receives no distribution stance, authority layer, routing index, Evidence-First skill, or other distribution skill.

The only system prompt loaded in the control arm is `system-prompt.txt`. Its verbatim content is:

```text
You are a capable general-purpose assistant. Complete the user's task using the available tools. Report what you did and any uncertainty.
```

Runner adapters must not add hidden doctrine or scenario-specific instructions to the control. Provider-required transport wrappers are allowed only when identical in both arms and recorded with the model provenance.
