# Improvement Proposal — This Repository

## Problem observed

Five overlapping public repositories and a later private capability estate require users to discover, combine, and maintain several systems. The existing consolidated super-repo has breadth but no measured admission gate, paired behavioral-delta evaluation, self-governance contract, or conditional supersession path.

## Evidence

The five source trees are frozen in `source-packages/README.md` at 475 tracked files. `docs/Why-This-Replaces-The-Super-Repo.md` records the inspected predecessor evidence. The dated plan, Opus review, Fable review, and binding amendments on `~/Desktop/` record the review sequence and its dispositions.

## Who should notice improvement

A new Hermes user performing a clean-room install; the repository maintainer reviewing one bounded release; and a future session routing to the right skill without loading the full catalog.

## Proposed destination

A new local Git repository intended for later operator publication and conditional replacement of the five predecessors.

## What it replaces or merges

It conditionally merges and replaces `second-brain-os-starter`, `second-brain-os-powerpack`, `second-brain-super-repo`, `agent-intelligence-economy`, and `agent-intelligence-context-pack`. They remain pinned sources until Release 1.0 passes all gates, a non-builder clean-room install succeeds, at least five real uses are recorded during the 30-day window, and the operator approves deprecation.

## Context, storage, and review cost

Release 1.0 is bounded to two shipped packs, two default operational skills, a routed skill library, a base vault, essential templates, paired evaluations, safe lifecycle scripts, and documentation. Phase 0 has a one-session budget; the full build has 15 builder sessions plus two operator review sessions. A phase that exceeds twice its estimate must stop and reduce scope.

## Blast radius

Public Git history, third-party licensing, privacy leakage, the user Hermes home and vault, predecessor status, and the attention of one maintainer.

## Verification method

Run clean-room install and real uninstall checks; paired treatment/control scenarios with three trials per arm; trace assertions; context-budget checks; worktree and sanitized-export history scans; and operator sanitization review. Deterministic treatment assertions must pass all three trials. Publication requires at least eight surviving scenarios where treatment passes all trials and control fails at least two trials, unless the operator signs a reasoned override.

## Kill or rollback condition

Stop and reduce scope if a phase exceeds twice its estimate. Do not publish without material paired-evaluation contribution under the fixed threshold or a signed operator override. Do not supersede predecessors without a successful non-builder clean-room install and five recorded real uses during the 30-day clock. Freeze expansion if no external user installs within 60 days. Archive rather than add packs if neither operator nor external downstream use is recorded within 90 days.

## Why a simpler location is insufficient

Extending the super-repo would preserve its vault-first breadth, overwrite-prone installation model, and ambiguous succession status. Extending the starter would recreate the starter-to-powerpack dependency chain. A new repository is justified only by enforceable self-governance, bounded scope, selective routing, behavioral controls, safe ownership, and explicit supersession conditions.

## Disposition

Act: proceed only with the bounded Release 1.0 and retain conditional supersession.
