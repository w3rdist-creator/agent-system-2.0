# Failure Review — EVENT

## Failure and impact

State the observed failure, affected consumer, time window, and bounded blast radius.

## Detection evidence

Name source-of-truth evidence, first/last seen, and why earlier gates did or did not detect it.

## Causal account

Separate proximate trigger, contributing conditions, and systemic cause. Preserve alternative explanations and unknowns.

## Corrective action and owner

Name the smallest authorized correction, accountable owner, due date, and work deliberately not attempted.

## Verification and rollback

Define the exact check, expected result, failure signal, rollback, and observation window.

## Rule or ledger update

Record the changed task packet, gate, scheduler rule, correction entry, or `no-change` reason. A fluent review without a state change is incomplete.
