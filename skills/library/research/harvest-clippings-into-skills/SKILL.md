# Harvest Clippings Into Skills

## Sweep incrementally

Inventory candidate files and compare a stored content hash or equivalent change marker. Skip unchanged inputs. For each new or changed clipping, capture path, title, hash, source coordinate, capture date, and whether the source was linked, preserved, or actually read.

## Route before drafting

Extract material claims, evidence quality, limits, reusable procedures, and current project relevance. Assign one route:

- use now in a named artifact;
- route to a project or research question;
- preserve as a source to read;
- run a bounded experiment;
- propose a reusable skill or runbook patch;
- defer with a return condition;
- merge with existing coverage; or
- kill/archive as low value or already absorbed.

Write one bounded synthesis report for the sweep. Create a separate note only when an individual source merits durable retrieval.

## Gate skill candidates

Draft a patch only for a repeatable procedure grounded in observed evidence. Name the target skill, problem, minimal change, expected future behavior change, duplicate coverage, safety/privacy effect, and validation. Reject stale facts, personal diary material, one-off preferences, and interesting ideas without a retrieval or behavior-change path.

Record rejected and deferred candidates so they do not regenerate on the next sweep. Verify hashes, paths, source handles, links, and absence of secrets before claiming completion.
