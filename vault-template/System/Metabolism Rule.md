# Metabolism Rule

Inbox is the only automatic write target for agents. A capture declares one shipped top-level destination with the `route:` frontmatter field. The metabolism command is the single mechanical owner of routing, exact-content deduplication, and decay pressure; it never deletes or overwrites content.

Scheduling remains operator-owned. Install at most one scheduler for this outcome and require a positive run heartbeat, following the single-scheduler ownership doctrine in the Agent Ops pack. Decayed captures enter [[Queues/Resolve Queue]] and remain in [[Inbox/Inbox Map]] until resolved.
